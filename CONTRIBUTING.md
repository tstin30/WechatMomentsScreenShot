### 🔧 代码贡献

#### 开发环境设置
```bash
# 克隆项目
git clone https://github.com/tstin30/WechatMomentsScreenShot.git
cd WechatMomentsScreenShot

# 直接用浏览器打开
open index.html
# 或使用本地服务器
python -m http.server 8000
```

#### 代码规范
- **JavaScript**: 使用ES6+语法，采用驼峰命名
- **CSS**: 使用BEM命名规范，保持代码整洁
- **HTML**: 语义化标签，合理的层级结构
- **注释**: 关键逻辑必须添加注释

#### 提交流程
1. **Fork项目** 到您的GitHub账号
2. **创建分支** `git checkout -b feature/amazing-feature`
3. **提交代码** `git commit -m 'feat: 添加新功能描述'`
4. **推送分支** `git push origin feature/amazing-feature`
5. **创建Pull Request**

#### Commit Message规范
```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

类型说明：
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档更新
- `style`: 代码格式调整（不影响功能）
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

示例：
```
feat(ui): 添加评论区居中显示功能

- 修改CSS样式实现文字居中
- 优化无评论时的空白处理
- 更新README文档说明

Closes #123
```

## 🎯 优先级

### 高优先级
- 🔥 **Bug修复** - 影响核心功能的问题
- 🔥 **性能优化** - 明显的用户体验改善
- 🔥 **安全相关** - 潜在的安全漏洞

### 中优先级
- ⚡ **功能增强** - 现有功能的改进
- ⚡ **兼容性** - 浏览器兼容性问题
- ⚡ **文档完善** - 文档错误或缺失

### 低优先级
- 💫 **新功能** - 全新功能需求
- 💫 **UI优化** - 界面美化类改动
- 💫 **重构** - 代码结构优化
